Forget the HTML and CSS if you are doing complex applications instead of traditional web pages. There is no need to construct new applications using archaic technologies that have evolved like the animals at the Chernobyl. As their original functions and purpose were lost long time ago they are now a grotesque freak show, abused and dysfunctional, only useful to serve as warning of man's folly.

It's time to leave that stuff behind and use something that can be actually used to build UI's instead of hacking and twisting HTML and CSS to fit your purposes. With Canvas and WebGL it is possible to build UI sytems completely inside the browser. Forget cross-broser incompabilities, pixel-perfect UI's are here and they are here to stay!

Why?
Portability, reliability, consistency and performance. UI rendering and layout computation is same to everyone.
Alternatives
You are either with canvas and WebGL or against us. There are no alternatives.
Extensibility
How do you extend HTML and CSS? That's right, you don't. With a JS framework you can bend it do anything you like and every browser will support it immediately.
